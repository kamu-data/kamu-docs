Kamu Node supports multiple network protocols for managing metadata, bulk uploads/downloads of datasets, and querying data.

Not sure where to start? Here are some suggestions:
- [REST](/node/api/rest) is best for interacting with the node from scripts, libraries, and system-to-system integration
- [GraphQL](/node/api/graphql) is best for interacting with the node when building rich applications
- [FlightSQL](/node/api/flight-sql) is the most modern way to connect SQL-based systems together including various BI and analytics tools
- [MQTT](/node/api/mqtt) is used to ingest real-time data from IoT devices
- [OData](/node/api/odata) can be used to integrate with some legacy systems
- [Blockchain Oracle](/node/api/oracle) can be used to query data directly from nodes into smart contracts
